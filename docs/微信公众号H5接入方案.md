# 元梦大陆 × 微信公众号 H5 接入方案

## 一、方案概述

将现有 Web 应用（`http://lucy.yinjoy.top/`）通过微信公众号以 H5 方式提供给用户访问，无需重写代码，零改造成本完成接入。用户在微信内点击公众号菜单即可打开完整的元梦大陆功能。

### 方案架构图

```
用户微信 → 公众号菜单/消息链接
              ↓
       微信内置浏览器
              ↓
    http://lucy.yinjoy.top/     ← 现有 Nginx（9999端口）
              ↓
    Vue3 前端（响应式 H5）
              ↓
    Express 后端 API（3001端口）
              ↓
         SQLite 数据库
```

## 二、前置条件

| 条件 | 要求 | 当前状态 |
|------|------|---------|
| 微信公众号 | 已认证的**订阅号**或**服务号** | 需申请 |
| 域名 HTTPS | 公众号内置浏览器强制要求 HTTPS | ⚠️ 当前为 HTTP，**需升级** |
| 备案 | 域名需完成 ICP 备案 | 需确认 |

## 三、HTTPS 升级（必须完成）

### 3.1 申请 SSL 证书（免费）

推荐使用阿里云免费 SSL 证书（个人版，有效期1年，可续签）：

1. 登录阿里云控制台 → **数字证书管理服务** → **SSL 证书**
2. 购买「个人测试证书（免费版）」
3. 填写域名 `lucy.yinjoy.top`，完成 DNS 验证
4. 下载证书文件（Nginx 格式）：得到 `cert.pem` 和 `cert.key`

### 3.2 上传证书到服务器

```bash
# 在服务器创建证书目录
mkdir -p /etc/nginx/ssl/lucy.yinjoy.top

# 上传证书文件（本地执行）
scp cert.pem root@服务器IP:/etc/nginx/ssl/lucy.yinjoy.top/
scp cert.key root@服务器IP:/etc/nginx/ssl/lucy.yinjoy.top/
```

### 3.3 修改 Nginx 配置支持 HTTPS

修改 Nginx 配置文件（通常在 `/etc/nginx/conf.d/yuanmeng.conf` 或 `/etc/nginx/nginx.conf`）：

```nginx
# HTTP → 强制跳转 HTTPS
server {
    listen 80;
    server_name lucy.yinjoy.top;
    return 301 https://$host$request_uri;
}

# HTTPS 主配置
server {
    listen 443 ssl;
    server_name lucy.yinjoy.top;

    # SSL 证书
    ssl_certificate     /etc/nginx/ssl/lucy.yinjoy.top/cert.pem;
    ssl_certificate_key /etc/nginx/ssl/lucy.yinjoy.top/cert.key;

    # SSL 安全配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    # 前端静态文件
    root /opt/yuanmeng-odyssey/frontend/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /api/ {
        proxy_pass http://localhost:3001/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # Session/Cookie 透传
        proxy_pass_header Set-Cookie;
        proxy_cookie_path / /;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|svg|ico|woff|woff2)$ {
        expires 7d;
        add_header Cache-Control "public, immutable";
    }
}
```

```bash
# 验证配置并重载
nginx -t && nginx -s reload
```

> **注意**：若当前 Nginx 监听在 9999 端口（非 80/443），需同步调整或在 443 端口新增 server 块。

## 四、微信公众号配置步骤

### 4.1 注册/登录公众号

访问 [https://mp.weixin.qq.com](https://mp.weixin.qq.com)，使用已认证的公众号账号登录。

> 推荐使用**服务号**（有自定义菜单、网页授权等更多能力），订阅号也可满足基本访问需求。

### 4.2 配置业务域名（JS-SDK / 网页授权）

**路径**：公众号后台 → **设置与开发** → **公众号设置** → **功能设置**

需要配置以下域名白名单（均填写 `lucy.yinjoy.top`）：

| 配置项 | 填写值 | 用途 |
|--------|--------|------|
| 业务域名 | `lucy.yinjoy.top` | 允许内置浏览器访问 |
| JS接口安全域名 | `lucy.yinjoy.top` | 若后续使用 JSSDK |
| 网页授权域名 | `lucy.yinjoy.top` | 若后续接入微信登录 |

**配置业务域名时**，微信会要求在服务器根目录放置一个验证文件：

```bash
# 微信提供一个类似 MP_verify_xxxxxxxx.txt 的文件
# 将其放到前端静态文件根目录
cp MP_verify_xxxxxxxx.txt /opt/yuanmeng-odyssey/frontend/dist/

# 验证地址：https://lucy.yinjoy.top/MP_verify_xxxxxxxx.txt
```

### 4.3 配置自定义菜单

**路径**：公众号后台 → **内容与互动** → **自定义菜单**

推荐菜单结构：

```
【元梦大陆】
├── 🏦 时光银行     → https://lucy.yinjoy.top/#/time-bank
├── 🌳 智慧树洞     → https://lucy.yinjoy.top/#/wisdom-tree  
└── ⚡ 数能充电站   → https://lucy.yinjoy.top/#/math-station
```

菜单配置方式：
1. 点击「添加菜单」→ 菜单名称填写（如「时光银行」）
2. 菜单内容选择「跳转网页」
3. 填入对应 URL（需为 HTTPS）
4. 发布菜单（可能需等待几分钟生效）

### 4.4 配置 IP 白名单（可选）

如需使用公众号消息推送能力：

**路径**：设置与开发 → **基本配置** → **IP白名单**

填入服务器公网 IP。

## 六、Session Cookie 跨域问题排查

### 问题背景

当前认证使用 Session + Cookie，微信内置浏览器对 Cookie 有以下限制：
- **SameSite=Lax**（默认）：跨域 POST 请求不携带 Cookie → 需设置为 `None` 或 `Lax`
- 若前后端同域（均为 `lucy.yinjoy.top`），Cookie 正常工作 ✅

### 当前架构验证

```
前端：https://lucy.yinjoy.top/         ← 同域
API： https://lucy.yinjoy.top/api/      ← 同域（Nginx代理）
```

由于前后端同域，Cookie 无跨域问题，**登录态可正常维持**。

### 后端 Session 配置建议

在 [backend/src/app.js](/Users/allen/workspaces/java/yuanmeng-odyssey/backend/src/app.js) 中确认 session 配置包含：

```javascript
app.use(session({
  secret: process.env.SESSION_SECRET || 'yuanmeng-secret',
  resave: false,
  saveUninitialized: false,
  cookie: {
    secure: true,        // HTTPS 环境必须为 true
    httpOnly: true,
    maxAge: 7 * 24 * 60 * 60 * 1000,  // 7天
    sameSite: 'lax'      // 微信内浏览器兼容
  }
}))
```

> 升级 HTTPS 后，必须将 `secure` 改为 `true`，否则 Cookie 不会被发送。
